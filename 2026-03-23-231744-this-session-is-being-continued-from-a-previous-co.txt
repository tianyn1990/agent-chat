⏺ 🎯 阶段六完整方案（基于 Star-Office-UI 深度调研）

  一、Star-Office-UI 项目分析总结

  ✅ 项目确认可用：
  - GitHub 地址：https://github.com/ringhyacinth/Star-Office-UI
  - 技术栈：Python Flask 后端 + 原生 JS + Phaser 3 游戏引擎
  - 项目大小：130MB（包含大量像素风图片资源）
  - 运行方式：独立服务，默认端口 19000
  - 通信方式：HTTP REST API（非 postMessage）

  核心 API：
  GET  /status          # 获取当前状态
  POST /set_state       # 设置状态
  GET  /agents          # 获取多 Agent 状态
  POST /agent-push      # Agent 主动推送状态
  GET  /yesterday-memo  # 获取昨日工作记录

  状态数据结构：
  {
    "state": "idle|writing|researching|executing|syncing|error",
    "detail": "状态描述文本",
    "progress": 0,
    "updated_at": "2026-03-23T16:00:00"
  }

  前端工作原理：
  - 每 2 秒轮询 /status 接口获取最新状态
  - 根据状态驱动像素角色移动到不同区域
  - 显示气泡对话框和状态文本
  - 支持多 Agent 协作显示

  ---
  二、集成方案重新评估

  ❌ 原文档方案（iframe + postMessage）不可行

  原因：
  1. Star-Office-UI 不支持 iframe 嵌入通信
  2. 前端是独立的 HTML 页面，不是可嵌入组件
  3. 通信方式是 HTTP REST API，不是 postMessage
  4. 需要独立部署后端服务（Flask + Python 3.10+）

  ✅ 可行的集成方案

  方案 A：新窗口打开（推荐）
  - 在 AI 消息上添加"查看工作流"按钮
  - 点击后 window.open('http://localhost:19000') 打开新标签页
  - 优点：简单、无侵入、用户体验好
  - 缺点：需要用户单独部署 Star-Office-UI

  方案 B：iframe 嵌入（有限支持）
  - 在侧边面板中嵌入 <iframe src="http://localhost:19000" />
  - 优点：集成在主应用内
  - 缺点：
    - 无法通过 postMessage 通信（Star-Office-UI 不支持）
    - 跨域问题（需要配置 CORS）
    - 性能开销（Phaser 游戏引擎）
    - 响应式适配问题（固定 1280x720）

  方案 C：状态同步（深度集成）
  - agent-chat 通过 HTTP API 向 Star-Office-UI 推送状态
  - 需要在 WebSocket 消息处理中添加状态映射逻辑
  - 优点：真正的状态同步
  - 缺点：需要后端支持、开发成本高

  🎯 深度分析：OpenClaw 多会话支持与我们的设计

  一、OpenClaw 的并发模型（关键发现）

  根据搜索结果，我找到了 OpenClaw 的核心并发机制：

  ✅ OpenClaw 支持多会话并发

  关键证据：

  1. "No two tasks from the same session can execute concurrently" (来源)
    - 同一会话内：任务串行执行（防止竞态条件）
    - 不同会话间：可以并发执行
  2. "Sessions are the backbone of OpenClaw's concurrency model" (来源)
    - 每个会话有独立的上下文、文件、历史记录
    - 会话之间完全隔离
  3. "Lane-aware FIFO queue" (来源)
    - 每个 session 有独立的 lane（通道）
    - 不同 lane 可以并发执行
    - 同一 lane 内串行执行
  4. "OpenClaw supports running multiple agents inside one Gateway process"
  (来源)
    - 支持多 Agent 同时运行
    - 每个 Agent 有独立的文件、内存、工具

  五、完整解决方案

  🎯 方案 A：单 Agent 全局状态可视化（推荐）

  核心思路：
  - Star-Office-UI 展示 Agent 的全局工作状态，而不是特定会话的状态
  - 当 Agent 在任意会话中工作时，都会更新全局状态
  - 用户看到的是"Agent 当前在做什么"，而不是"Agent 在哪个会话中做什么"

  // WebSocket 消息处理
  onMessage(message) {
    const { sessionId, type, content } = message;

    } else if (type === 'assistant_message') {
      state = 'writing';
      detail = '正在生成回复';
    } else if (type === 'chart') {
      state = 'executing';
      detail = '正在生成图表';
    } else if (type === 'error') {
      state = 'error';
      detail = content.message;
    }

    // 推送到 Star-Office-UI（不带 sessionId）
    await fetch('http://star-office-ui:19000/agent-push', {
      method: 'POST',
      body: JSON.stringify({
        agentId: 'openclaw_main',
        joinKey: process.env.STAR_OFFICE_JOIN_KEY,
        state,
        detail,
      }),
    });
  }
