## MODIFIED Requirements

### Requirement: 沉浸式工作台外层视觉必须与主产品统一

系统 MUST 在保持沉浸式 `iframe-first` 工作台结构的前提下，使工作台外层控件与主产品视觉系统统一；同时外层控件 MUST 更扁平、更轻量，并以不遮挡真实像素办公室为优先原则。

#### Scenario: 用户进入沉浸式工作台

- **Given** 用户从聊天页打开某会话的执行状态工作台
- **When** 沉浸式工作台展示真实 `Star-Office-UI` iframe
- **Then** 外层壳层必须继续保持极简
- **And** 外层控件不得重新铺设大面积标题栏、说明栏或状态面板
- **And** 用户必须能够感知这是同一产品中的沉浸式场景

### Requirement: 工作台控件必须兼顾可发现性与低遮挡

系统 MUST 为沉浸式工作台提供清晰可发现的返回与辅助操作控件，但这些控件 MUST 优先贴近外层边缘布置，避免遮挡像素办公室核心可视区域。

#### Scenario: 用户需要离开工作台

- **Given** 用户正在查看沉浸式工作台
- **When** 用户希望返回聊天界面
- **Then** 返回入口必须在无需依赖浏览器原生返回按钮的情况下可直接发现和点击
- **And** 该入口不得被放置在容易与 iframe 内原生控件混淆的位置

#### Scenario: 用户需要使用次级操作

- **Given** 用户在沉浸式工作台中需要刷新或收起工作台
- **When** 用户查看外层控件区
- **Then** 次级操作必须可访问
- **And** 其存在感不得高于主返回操作
- **And** 外层控件整体不得对真实办公室内容造成明显遮挡
