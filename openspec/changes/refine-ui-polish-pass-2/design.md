# 设计文档：UI 细节打磨（第二轮）

## Context

第一轮升级已经完成统一视觉系统，但在真实使用场景中暴露了若干“可感知但非功能性”的体验问题。此类问题若不修复，会削弱对整体视觉方案的认可度。

## Goals / Non-Goals

### Goals

- 降低聊天页标题区的压迫感与信息噪音
- 优化首屏欢迎态与输入区的可见性与操作引导
- 修复技能页信息冗余，强化信息架构清晰度
- 修正登录页 heading 结构并提升 CTA 视觉权重
- 补齐可访问性 label

### Non-Goals

- 不修改核心功能路径、路由与状态模型
- 不引入新的 UI 框架
- 不新增业务模块

## Decisions

### Decision: 标题区遵循“主标题优先”

- 取消原始 `sessionId` 直出
- meta 卡片缩小并收敛宽度
- 允许标题 balanced wrap，避免单词孤立

### Decision: 技能页保持信息分区但避免重复

- 已安装技能仍单独展示
- “所有技能”默认排除已安装项
- 当过滤条件仅命中已安装项时，提示用户无需重复展示

### Decision: 登录页主 CTA 必须成为视觉焦点

- 保留左侧品牌叙事，但减少其压迫感
- 右侧卡片的 CTA 位置与层级权重优先

## Risks / Trade-offs

- 过度压缩 meta 可能导致信息不足
  - 通过保持 2~3 个摘要点而不展示技术 ID 来平衡

- 已安装与所有技能分区后可能让“所有技能”列表显得变短
  - 通过提示文案与栅格布局调整减轻落差

## Migration Plan

1. 收敛 Chat 标题区、欢迎态与 message rail
2. 调整 Skills 信息架构与卡片展示
3. 修正 Login heading 与 CTA 权重
4. 补齐 aria-label 与测试
