# 阶段二：飞书登录模块

## 1. 目标
- 实现飞书 OAuth 2.0 授权登录流程
- 获取并展示用户信息（头像、姓名、部门）
- 实现 Token 管理（存储、刷新、过期处理）
- 登录状态持久化，未登录跳转至登录页

## 2. 飞书 OAuth 2.0 登录流程

```
用户访问应用
    │
    ▼
检查本地 Token
    │
    ├─ 有效 ──→ 进入主界面
    │
    └─ 无效 ──→ 跳转登录页
                    │
                    ▼
              点击"飞书登录"
                    │
                    ▼
           重定向到飞书授权页
           (携带 client_id、redirect_uri、state)
                    │
                    ▼
           用户在飞书完成授权
                    │
                    ▼
           回调 redirect_uri
           (携带 code、state)
                    │
                    ▼
         前端将 code 发送给后端
                    │
                    ▼
         后端换取 access_token
         并获取用户信息
                    │
                    ▼
         返回 JWT + 用户信息给前端
                    │
                    ▼
         前端存储 Token，进入主界面
```

### 3.2 Token 管理策略
- **存储**: `localStorage`，key 为 `oc_token`
- **格式**: JWT，包含过期时间 `exp`
- **刷新**: 每次请求前检查过期时间，剩余 5 分钟内自动刷新
- **过期处理**: Token 过期后清空本地存储，跳转登录页

## 6. 关键接口定义（与后端约定）
```typescript
// POST /api/auth/feishu/callback
// 前端将飞书返回的 code 发给后端
Request: { code: string; state: string }
Response: { token: string; userInfo: UserInfo }

// GET /api/auth/me
// 获取当前用户信息（需带 Authorization header）
Response: { userInfo: UserInfo }

// POST /api/auth/refresh
// 刷新 Token
Request: { token: string }
Response: { token: string }

// POST /api/auth/logout
// 登出
Response: { success: boolean }
```
