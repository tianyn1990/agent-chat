# OpenClaw Web 客户端 - 总体设计方案

## 1. 项目概述

### 1.1 项目背景
为公司私有部署的 OpenClaw 开发一个现代化的 Web 交互界面，提供完整的对话、技能管理、可视化等功能。

### 1.2 核心目标
- 提供流畅的 AI 对话体验
- 支持丰富的内容展示（文本、卡片、图表、图片）
- 集成飞书登录，保障企业安全
- 提供技能市场，扩展 OpenClaw 能力
- 可视化展示 OpenClaw 工作流程

## 2. 技术栈选型

### 2.1 核心技术
- **前端框架**: React 18+ (Hooks + TypeScript)
- **构建工具**: Vite 5+
- **UI 组件库**: Ant Design 5+
- **状态管理**: Zustand (轻量、简单、TypeScript 友好)
- **路由**: React Router 6+
- **样式方案**: CSS Modules + Less

### 2.2 数据可视化
- **图表库**: AntV 系列
  - G2: 统计图表（折线图、柱状图、饼图等）
  - G6: 关系图、流程图
  - S2: 表格（支持复杂表格场景）

### 2.3 通信协议
- **实时通信**: WebSocket (原生 WebSocket API)
- **HTTP 请求**: Axios
- **认证**: OAuth 2.0 (飞书开放平台)

### 2.4 开发工具
- **代码规范**: ESLint + Prettier
- **类型检查**: TypeScript 5+
- **测试框架**: Vitest + React Testing Library
- **Git Hooks**: Husky + lint-staged

## 4. 核心功能设计

### 4.1 飞书登录模块
**功能描述**:
- OAuth 2.0 授权登录
- 获取用户信息（头像、姓名、部门）
- Token 管理（存储、刷新、过期处理）
- 登录状态持久化

**技术实现**:
- 使用飞书开放平台 OAuth 2.0 流程
- LocalStorage 存储 Token
- Axios 拦截器自动添加 Token
- 路由守卫保护需要登录的页面

### 4.2 对话模块
**功能描述**:
- 多会话管理（创建、删除、切换）
- 实时消息收发
- 消息类型支持：
  - 文本消息
  - 交互式卡片
  - 图表（图片、表格、统计图）
  - 文件上传
- 虚拟滚动（长对话性能优化）
- 自动滚动到最新消息
- 消息历史加载

**技术实现**:
- WebSocket 实时通信
- 虚拟列表优化长列表性能
- 消息队列处理并发消息
- 乐观更新提升用户体验

### 4.3 交互式卡片
**功能描述**:
- 支持多种交互组件：
  - 按钮（单选、多选）
  - 表单（输入框、下拉框、日期选择器）
  - 图片、链接
- 卡片状态管理
- 用户操作回传

**卡片协议设计** (简化版飞书卡片):
```json
{
  "type": "interactive_card",
  "header": {
    "title": "卡片标题",
    "template": "blue"
  },
  "elements": [
    {
      "tag": "div",
      "text": "描述文本"
    },
    {
      "tag": "action",
      "actions": [
        {
          "tag": "button",
          "text": "确认",
          "type": "primary",
          "value": "confirm"
        }
      ]
    }
  ]
}
```

### 4.4 图表模块
**功能描述**:
- 图片展示（支持预览、下载）
- 表格展示（AntV S2，支持排序、筛选）
- 统计图表（AntV G2，支持折线图、柱状图、饼图等）
- 关系图（AntV G6，支持流程图、知识图谱）

**图表协议设计**:
```json
{
  "type": "chart",
  "chartType": "line|bar|pie|table|image|graph",
  "data": {},
  "config": {}
}
```

**方案 A（主方案）**: Web 端动态渲染
- OpenClaw 返回标准化的图表配置 JSON
- Web 端使用 AntV 组件渲染
- 支持交互（缩放、筛选、导出）

**方案 B（备选）**: iframe 嵌入
- OpenClaw 生成完整 HTML 页面
- 发布为独立站点
- Web 端 iframe 嵌入
- 提供"在新窗口打开"功能

### 4.5 技能市场
**功能描述**:
- 技能列表展示（卡片式布局）
- 技能详情查看
- 技能安装/卸载
- 技能分类筛选
- 技能搜索

**数据结构**:
```typescript
interface Skill {
  id: string;
  name: string;
  description: string;
  icon: string;
  category: string;
  version: string;
  author: string;
  installed: boolean;
  config?: Record<string, any>;
}
```

### 4.6 可视化模块（Star-Office-UI 集成）
**功能描述**:
- 展示 OpenClaw / Agent 执行状态
- 与当前会话联动展示执行上下文
- 通过服务端适配层接入 Star-Office-UI
- 为后续工作流图扩展预留空间

**集成方式**:
- 将 Star-Office-UI 作为独立服务部署
- 通过 iframe 嵌入到主应用
- 由服务端适配层完成 session 到状态看板的翻译
- 当前阶段不依赖 postMessage

## 5. WebSocket 协议设计

### 5.1 连接管理
```typescript
// 连接 URL
ws://[host]/ws?token=[auth_token]

// 心跳机制
每 30 秒发送 ping，超时 60 秒断开重连
```

### 5.2 消息格式
```typescript
// 客户端 -> 服务端
{
  "type": "user_message" | "card_action" | "file_upload",
  "sessionId": "string",
  "content": "string | object",
  "timestamp": "number"
}

// 服务端 -> 客户端
{
  "type": "assistant_message" | "card" | "chart" | "error",
  "sessionId": "string",
  "content": "string | object",
  "timestamp": "number",
  "messageId": "string"
}
```

### 5.3 消息类型定义
- `user_message`: 用户文本消息
- `assistant_message`: AI 回复消息
- `card`: 交互式卡片
- `card_action`: 用户卡片操作
- `chart`: 图表数据
- `file_upload`: 文件上传
- `error`: 错误消息
- `session_created`: 会话创建
- `session_deleted`: 会话删除

## 7. 非功能性需求

### 7.1 性能要求
- 首屏加载时间 < 2s
- 消息发送响应 < 100ms
- 长列表滚动流畅（60fps）
- 支持 1000+ 条消息的会话

### 7.2 兼容性
- Chrome 90+
- Edge 90+
- Firefox 88+
- Safari 14+

### 7.3 安全性
- HTTPS 通信
- Token 加密存储
- XSS 防护
- CSRF 防护

### 7.4 可维护性
- TypeScript 类型覆盖率 > 90%
- 单元测试覆盖率 > 70%
- 代码注释覆盖关键逻辑
- 组件文档完善

## 8. 风险与挑战

### 8.1 技术风险
- **WebSocket 稳定性**: 需要完善的重连机制和错误处理
- **长列表性能**: 使用虚拟滚动优化
- **图表渲染性能**: 大数据量时需要降采样或分页

### 8.2 业务风险
- **OpenClaw 协议变更**: 设计灵活的协议适配层
- **飞书 API 限流**: 实现请求队列和重试机制

### 8.3 解决方案
- 完善的错误处理和降级方案
- 充分的单元测试和集成测试
- 详细的开发文档和注释
