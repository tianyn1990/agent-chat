## 1. 目标
- 实现多会话管理（创建、删除、重命名、切换）

- 消息列表展示（文本、Loading 状态、错误状态）

- 自动滚动到最新消息 + 手动滚动时暂停自动滚动
- 输入框支持文件上传、快捷键发送

## 3. WebSocket 通信协议（完整定义）

### 3.1 连接
```
WebSocket URL: ws://{host}/ws
连接时携带 Authorization header（或通过 query 参数 token）

握手后服务端发送：
{
  "type": "connected",
  "data": { "serverId": "string", "timestamp": number }
}
```

### 3.2 心跳
```typescript
// 客户端每 30s 发送
{ "type": "ping", "timestamp": number }

// 服务端回复
{ "type": "pong", "timestamp": number }
```

### 3.3 消息类型总览

#### 客户端 → 服务端
```typescript
// 用户发送文本消息
{
  "type": "user_message",
  "sessionId": "string",
  "requestId": "string",   // 前端生成的唯一请求ID，用于对应响应
  "content": {
    "text": "string",
    "files"?: [            // 可选，文件上传时携带
      { "fileId": "string", "fileName": "string", "fileType": "string" }
    ]
  }
}

// 用户操作交互卡片
{
  "type": "card_action",
  "sessionId": "string",
  "requestId": "string",
  "cardId": "string",      // 卡片的唯一ID
  "action": {
    "tag": "button" | "select" | "input",
    "key": "string",       // 元素标识
    "value": "any"         // 用户选择/输入的值
  }
}

// 创建会话
{
  "type": "create_session",
  "requestId": "string",
  "title"?: "string"
}

// 删除会话
{
  "type": "delete_session",
  "sessionId": "string"
}
```

#### 服务端 → 客户端
```typescript
// 文本流式回复（streaming）
{
  "type": "message_chunk",
  "sessionId": "string",
  "messageId": "string",
  "requestId": "string",
  "delta": "string",       // 本次增量文本
  "done": boolean          // 是否是最后一块
}

// 交互式卡片消息
{
  "type": "card",
  "sessionId": "string",
  "messageId": "string",
  "card": { /* 卡片 JSON，见阶段四 */ }
}

// 图表消息
{
  "type": "chart",
  "sessionId": "string",
  "messageId": "string",
  "chart": { /* 图表 JSON，见阶段四 */ }
}

// 会话创建成功
{
  "type": "session_created",
  "requestId": "string",
  "session": {
    "id": "string",
    "title": "string",
    "createdAt": number
  }
}

// 错误消息
{
  "type": "error",
  "sessionId"?: "string",
  "requestId"?: "string",
  "code": "string",
  "message": "string"
}
```

## 4. 核心数据类型

```typescript
// src/types/message.ts

/** 消息发送方 */
type MessageRole = 'user' | 'assistant';

/** 消息内容类型 */
type MessageContentType = 'text' | 'card' | 'chart' | 'file' | 'error';

/** 消息状态 */
type MessageStatus = 'sending' | 'streaming' | 'done' | 'error';

interface Message {
  id: string;
  sessionId: string;
  role: MessageRole;
  contentType: MessageContentType;
  content: TextContent | CardContent | ChartContent | FileContent;
  status: MessageStatus;
  timestamp: number;
}

interface TextContent {
  text: string;
}

interface FileContent {
  fileId: string;
  fileName: string;
  fileType: string;
  fileSize: number;
  previewUrl?: string;
}

// src/types/session.ts
interface Session {
  id: string;
  title: string;
  lastMessage?: string;     // 最后一条消息摘要
  lastMessageAt?: number;
  createdAt: number;
  messageCount: number;
}
```

## 8. 消息列表滚动策略
```
1. 新消息到达时：
   - 若用户在底部附近（距底部 < 100px）→ 自动滚动到底部
   - 若用户已向上滚动 → 不自动滚动，显示"↓ 新消息"按钮

2. 加载历史消息：
   - 滚动到顶部时触发加载
   - 保持当前滚动位置（锚定到原来的第一条消息）

3. 虚拟滚动：
   - 使用 @tanstack/react-virtual 或自实现
   - 仅渲染视口内的消息
   - 预估消息高度，动态更新
```

## 9. 文件上传流程
```
1. 用户点击 📎 或拖拽文件到输入框
2. 前端调用 POST /api/upload 上传文件，获取 fileId
3. 发送消息时携带 fileId
4. 服务端处理文件并发送给 OpenClaw
5. 前端显示文件预览（图片直接预览，其他显示文件名+大小）
```
